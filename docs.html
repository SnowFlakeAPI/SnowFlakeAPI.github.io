Introduction

Welcome to the Snowflake API documentation. Snowflake is a secure and reliable key system designed for simplicity.

Base URL

https://yourdomain.com/v1

Features

authentication · validation · management · hwid binding

Authentication

All API requests require authentication using your Snowflake API key. Include your API key in the request header.

Header

Authorization: Bearer YOUR_SNOWFLAKE_API_KEY

Getting Your Snowflake API Key

You need to be a registered user to obtain a Snowflake API key. First, log in to your account, then request an API key through the /api/snowflake/apikey endpoint. Keep your API key secure and never share it publicly.

Get API Key

Create a Snowflake API key for your account. Requires user authentication token.

POST /api/snowflake/apikey

Headers

Authorization: Bearer YOUR_USER_AUTH_TOKEN

Body Parameters

ParameterTypeDescription
planstringPlan tier: free, basic, pro, enterprise (default: free)

Response

{
  "success": true,
  "apiKey": "sfk_a1b2c3d4...",
  "plan": "free",
  "created": "2025-01-15T10:30:00.000Z",
  "message": "API key created successfully"
}

List Your API Keys

GET /api/snowflake/apikeys

Delete API Key

DELETE /api/snowflake/apikey/{apiKey}

Quick Start

Get started with Snowflake API in minutes.

Step 1: Get Your Snowflake API Key

Register an account and create a Snowflake API key via POST /api/snowflake/apikey

Step 2: Make Your First Request

// validate a key
GET /v1/validate?key=SNOW-XXXX-XXXX

Step 3: Handle Response

{
  "valid": true,
  "key": "SNOW-XXXX-XXXX",
  "expires": "2025-12-31",
  "hwid": null,
  "uses": 1
}

Validate Key

Check if a license key is valid and retrieve its metadata.

GET /v1/validate?key={key}&hwid={hwid}

Headers

Authorization: Bearer YOUR_SNOWFLAKE_API_KEY

Query Parameters

ParameterTypeDescription
keystringThe license key to validate (required)
hwidstringHardware ID for binding (optional)

Response

{
  "valid": true,
  "key": "SNOW-XXXX-XXXX",
  "expires": "2025-12-31",
  "hwid": "ABC123",
  "uses": 5
}

Create Key

Generate a new license key.

POST /v1/create

Headers

Authorization: Bearer YOUR_SNOWFLAKE_API_KEY
Content-Type: application/json

Body Parameters

ParameterTypeDescription
durationintegerKey duration in days (required)
notestringOptional note for the key
hwid_lockbooleanEnable hardware ID binding (default: false)

Request Body

{
  "duration": 365,
  "note": "Premium user",
  "hwid_lock": true
}

Response

{
  "success": true,
  "key": "SNOW-A1B2-C3D4",
  "expires": "2025-12-31",
  "created": "2025-01-01",
  "hwidLock": true
}

Rate Limits

API requests are rate limited to ensure fair usage. Rate limits are based on your Snowflake API key plan.

PlanRequestsWindow
Free100per minute
Basic500per minute
Pro2000per minute
EnterpriseUnlimited-

Rate Limit Headers

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1704067200

Rate limits reset every minute. The X-RateLimit-Reset header indicates the Unix timestamp when your limit will reset.

Examples

Code examples for common use cases.

Python

import requests

url = "https://yourdomain.com/v1/validate"
headers = {"Authorization": "Bearer YOUR_SNOWFLAKE_API_KEY"}
params = {"key": "SNOW-XXXX-XXXX"}

response = requests.get(url, headers=headers, params=params)
print(response.json())

JavaScript

const response = await fetch(
  "https://yourdomain.com/v1/validate?key=SNOW-XXXX-XXXX",
  { headers: { "Authorization": "Bearer YOUR_SNOWFLAKE_API_KEY" } }
);

const data = await response.json();
console.log(data);

cURL

curl -X GET "https://yourdomain.com/v1/validate?key=SNOW-XXXX-XXXX" \
  -H "Authorization: Bearer YOUR_SNOWFLAKE_API_KEY"

Creating a Key

curl -X POST "https://yourdomain.com/v1/create" \
  -H "Authorization: Bearer YOUR_SNOWFLAKE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"duration": 365, "note": "Test key", "hwid_lock": true}'
